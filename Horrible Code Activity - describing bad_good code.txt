Good Code (Lauren Bell, Shane Moua, Mika Klymenko): 
        Principles used
1. KISS/DRY - We simplified the original bad code by removing unnecessary lines in both the addition and subtraction functions, specifically the variables x and thisistheresult, as they were not needed for the function to work as efficiently as possible. We also removed the loop in the multiply() function, updating every function to include only 1 line of code, ensuring the program is  as “dry” and simple as possible.

3. Single Responsibility -  The Calculator class originally inherited the ScientificCalculator class, which would be incorrect as the ScientificCalculator should be the one inheriting the Calculator class (a simple program). This could lead to an explosion of other subclasses and issues, making the program useless. We removed the inheritance variable entirely to ensure composition over inheritance.
